# Gestión de la Seguridad de la Información - Familia ISO/IEC 27000

## Marco de Referencia
Para garantizar la protección de los activos de información de **Indurama** y sus proveedores, el sistema adopta controles y directrices basados en la familia de normas **ISO/IEC 27000**.

## ISO/IEC 27001 (SGSI)
El sistema apoya la implementación de un Sistema de Gestión de Seguridad de la Información (SGSI) mediante:
* **Confidencialidad:** Asegurando que la información sensible (datos bancarios, tributarios) sea accesible solo por personal autorizado (Roles: Gestor, Administrador).
* **Integridad:** Garantizando que la información de los proveedores no sea modificada sin autorización.
* **Disponibilidad:** Asegurando el acceso continuo a la plataforma Web y Móvil mediante arquitectura en la nube.

## ISO/IEC 27002 (Controles de Seguridad)
Se implementan los siguientes dominios de control aplicados al desarrollo de software:

### 1. Control de Acceso (A.9)
* Uso de autenticación robusta para usuarios móviles y web.
* Gestión de privilegios basada en roles (RBAC) para diferenciar entre *Solicitantes*, *Gestores* y *Proveedores*.

### 2. Criptografía (A.10)
* **En Tránsito:** Uso obligatorio de HTTPS (TLS 1.2+) para todas las comunicaciones entre la App Móvil, Web y el Backend.
* **En Reposo:** Cifrado de datos sensibles en la base de datos (ej. contraseñas, tokens bancarios).

### 3. Seguridad en las Operaciones (A.12)
* Separación de entornos de desarrollo, pruebas y producción.
* Respaldos automatizados de la base de datos de proveedores.

### 4. Seguridad en las Comunicaciones (A.13)
* Segregación de redes y uso de Firewalls de Aplicación Web (WAF) para proteger las APIs.
