# Garantía de Seguridad Técnica y Aplicada (Entorno Firebase)

## Responsabilidad
La seguridad del aplicativo es gestionada por el rol de **Security Developer** (Edison Chamba), encargado de la configuración de reglas en la consola de Firebase y pruebas de vulnerabilidad.

## Estrategias de Implementación en Firebase

### 1. Autenticación y Autorización (Firebase Auth)
* **Gestión de Identidad:** Uso de **Firebase Authentication** para manejar el ciclo de vida de los usuarios (registro, login, recuperación).
* **Claims Personalizados:** Implementación de "Custom Claims" en los tokens de usuario para diferenciar roles (*Solicitante*, *Gestor*, *Proveedor*) y restringir el acceso a datos sensibles desde el nivel de autenticación.

### 2. Protección de Datos (Firestore Security Rules)
En lugar de prevenciones tradicionales de SQL, utilizamos el motor de reglas de Firestore para validar cada lectura y escritura:
* **Validación de Esquema:** Las reglas aseguran que los datos entrantes tengan el formato correcto (ej. que el campo 'costo' sea un número y no un string).
* **Control de Acceso Granular:** Reglas estrictas que permiten, por ejemplo, que un *Proveedor* solo pueda leer y editar su propio documento de perfil (`request.auth.uid == resource.data.uid`).
* **Prevención de Inyección NoSQL:** Al utilizar las SDKs nativas de Firebase y evitar la construcción de consultas mediante concatenación de strings en Cloud Functions, mitigamos riesgos de inyección.

### 3. Seguridad del Cliente y API
* **Firebase App Check:** Implementación de App Check para asegurar que las peticiones al backend provengan genuinamente de nuestra app oficial (iOS/Android/Web) y no de bots o scripts maliciosos.
* **Cifrado:** Firebase gestiona automáticamente el cifrado en tránsito (HTTPS) y el cifrado en reposo de la base de datos.

### 4. Evaluación de Riesgos (AMEF)
Como parte del módulo EPI (Evaluación de Proveedor Interno), se integra la metodología **AMEF (Análisis de Modo y Efecto de Falla)** para evaluar riesgos operativos y de calidad asociados a cada proveedor antes de su contratación.
