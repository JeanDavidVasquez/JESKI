# Garantía de Seguridad Técnica y Aplicada

## Responsabilidad
La seguridad del aplicativo es gestionada por el rol de **Security Developer** (Edison Chamba), encargado de la implementación de defensas y pruebas de vulnerabilidad.

## Estrategias de Implementación

### 1. Autenticación y Autorización
* **JWT (JSON Web Tokens):** Implementación de tokens firmados para la gestión de sesiones sin estado (stateless) en la API REST.
* **OAuth 2.0:** Protocolo estándar utilizado para integraciones seguras y validación de identidad.

### 2. Protección contra Amenazas (OWASP Top 10)
Se aplican medidas preventivas contra las vulnerabilidades más comunes:
* **Inyección SQL:** Uso de ORMs y consultas parametrizadas para evitar la manipulación de base de datos.
* **XSS (Cross-Site Scripting):** Sanitización de entradas en el Frontend y Backend para evitar la ejecución de scripts maliciosos.
* **Exposición de Datos Sensibles:** Enmascaramiento de datos fiscales y bancarios en las vistas no privilegiadas.

### 3. Seguridad en la Nube
El despliegue en la nube incluye:
* Grupos de seguridad configurados para permitir tráfico solo en puertos estrictamente necesarios (443).
* Monitoreo de logs de acceso y auditoría de cambios en los registros de proveedores.

### 4. Evaluación de Riesgos (AMEF)
Como parte del módulo EPI (Evaluación de Proveedor Interno), se integra la metodología **AMEF (Análisis de Modo y Efecto de Falla)** para evaluar riesgos operativos y de calidad asociados a cada proveedor antes de su contratación.
